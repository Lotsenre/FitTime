# Техническое задание
## FitTime — Информационная система для фитнес-центра
### Расписание занятий и управление абонементами

---

## 1. Общее описание

**Название системы:** FitTime
**Тип приложения:** Десктопное Windows-приложение
**Целевая аудитория:** Административный персонал и тренеры фитнес-центра
**Цель:** Автоматизация управления клиентами, абонементами, расписанием занятий и посещаемостью

---

## 2. Фирменный стиль

### Цветовая палитра

| Название | HEX | Применение |
|----------|-----|------------|
| Чёрный | `#000000` | Основной фон, боковое меню, заголовки |
| Оранжевый | `#F4B448` | Акцентный цвет: кнопки, иконки, выделения |
| Белый | `#FFFFFF` | Фон карточек, текст на тёмном фоне |

### Типографика

| Параметр | Значение |
|----------|----------|
| Шрифт | Roboto |
| Заголовки | Roboto Medium / Bold |
| Основной текст | Roboto Regular, 14pt |
| Подписи и метки | Roboto Light, 12pt |

> Roboto — стандартный шрифт Material Design, поставляется вместе с MaterialDesignInXamlToolkit. Подключение не требует дополнительных действий.

### Общий стиль интерфейса
- Тёмная тема (Dark Theme) в MaterialDesignInXamlToolkit
- Акцентный цвет: оранжевый `#F4B448`
- Боковое меню (NavigationDrawer): чёрный фон, белые иконки и текст, активный пункт подсвечивается оранжевым
- Кнопки действий (primary): оранжевый фон, чёрный текст
- Карточки занятий в календаре: цвет берётся из поля `Color` таблицы `ClassTypes`

## 3. Технический стек

| Компонент | Технология |
|-----------|-----------|
| Язык программирования | C# (.NET 8) |
| UI-фреймворк | WPF (Windows Presentation Foundation) |
| Архитектурный паттерн | MVVM (Model–View–ViewModel) |
| ORM | Entity Framework Core 8 (без миграций) |
| База данных | PostgreSQL |
| EF Core провайдер | Npgsql.EntityFrameworkCore.PostgreSQL |
| UI-библиотека | MaterialDesignInXamlToolkit |
| Навигация | MVVM Community Toolkit |
| Экспорт данных | CsvHelper |
| Логирование | Serilog |
| Контейнер зависимостей | Microsoft.Extensions.DependencyInjection |

> Инициализация базы данных производится вручную через SQL-скрипт `fittime_db.sql`. EF Core используется только для выполнения запросов.

**Структура проекта (Solution):**

```
FitTime.sln
└── FitTime/                        # Единственный WPF-проект
    ├── Models/                     # Классы сущностей (Client, User, Membership и др.)
    ├── ViewModels/                 # ViewModel для каждого экрана
    ├── Views/                      # XAML-окна и пользовательские элементы управления
    ├── Data/                       # DbContext и репозитории
    ├── Services/                   # Бизнес-логика (абонементы, посещения, отчёты)
    ├── Helpers/                    # Вспомогательные классы, конвертеры XAML
    ├── Assets/                     # Иконки и изображения
    ├── appsettings.json            # Строка подключения к PostgreSQL
    └── App.xaml                    # Точка входа, настройка DI-контейнера
```

## 5. Функциональные требования

### 5.1 Модуль авторизации
- Вход по логину и паролю (пароли хранятся в виде BCrypt-хэша)
- Разграничение доступа по роли
- Блокировка аккаунта после 5 неудачных попыток входа
- Смена пароля

### 5.2 Модуль управления клиентами
- Добавление, редактирование, мягкое удаление клиентов (IsActive = false)
- Поиск по ФИО, телефону, email
- Просмотр профиля клиента (личные данные + история абонементов + посещаемость)
- Фильтрация клиентов по статусу абонемента (активный / просроченный / нет абонемента)

### 5.3 Модуль абонементов
- Справочник типов абонементов (название, длительность в днях, кол-во посещений, цена)
- Поддержка безлимитных абонементов
- Оформление и продление абонемента для клиента
- Автоматический контроль срока действия и остатка посещений
- История абонементов клиента
- Уведомление о скором истечении абонемента (за 7 дней)

### 5.4 Модуль расписания занятий
- Создание и редактирование занятий (тип, тренер, дата/время, макс. кол-во участников)
- Каждое занятие создаётся отдельно, без повторений
- Визуальный недельный календарь занятий
- Запись клиента на занятие менеджером
- Проверка на пересечение занятий у клиента (нельзя записать на два занятия в одно время)
- Блокировка записи при отсутствии активного абонемента
- Блокировка записи при превышении числа участников

### 5.5 Модуль учёта посещений
- Отметка посещения клиента на занятии (тренером или менеджером)
- Списание одного визита с абонемента в момент отметки (не при записи)
- Отмена посещения с возвратом визита
- Журнал посещений с фильтрами по дате, клиенту, занятию

### 5.6 Модуль тренеров
- Управление тренерами (ФИО, телефон, email, специализация)
- Тренер обязательно имеет аккаунт пользователя в системе
- Просмотр расписания конкретного тренера
- Рабочая нагрузка тренера (кол-во занятий за период)

### 5.7 Модуль отчётов
- Отчёт по посещаемости за период
- Отчёт по продажам абонементов за период
- Список клиентов с истекающими абонементами
- Загруженность тренеров
- Экспорт отчётов в CSV через CsvHelper

## 6. Описание основных форм и диалогов

### 6.1 Форма входа (`LoginWindow`)
**Поля:** Логин, Пароль
**Кнопки:** Войти
**Поведение:** При успехе — открывается `MainWindow`, при неверных данных — сообщение об ошибке

---

### 6.2 Главное окно (`MainWindow`)
**Структура:** Боковое меню (NavigationDrawer) + область контента
**Пункты меню:**
- Панель управления (дашборд)
- Клиенты
- Абонементы
- Расписание
- Тренеры
- Посещения
- Отчёты
- Пользователи *(только Admin)*

**Дашборд содержит:**
- Кол-во активных клиентов
- Кол-во занятий сегодня
- Кол-во истекающих абонементов (ближайшие 7 дней)

---

### 6.3 Список клиентов (`ClientsView`)
**Элементы:**
- Строка поиска (по ФИО, телефону, email)
- Фильтр по статусу абонемента
- DataGrid с колонками: ФИО, Телефон, Email, Статус абонемента, Дата регистрации
- Кнопки: Добавить, Редактировать, Деактивировать, Открыть профиль

---

### 6.4 Диалог клиента (`ClientDialog`)
**Вкладки:**
- **Личные данные:** Имя, Фамилия, Отчество, Дата рождения, Телефон, Email, Заметки
- **Абонементы:** История абонементов, кнопка «Оформить абонемент»
- **Посещения:** Журнал посещений клиента

---

### 6.5 Диалог оформления абонемента (`MembershipDialog`)
**Поля:**
- Клиент (выбор из списка или заполнен автоматически)
- Тип абонемента (выпадающий список с ценой и описанием)
- Дата начала (DatePicker)
- Дата окончания (вычисляется автоматически)
- Кол-во оставшихся посещений (заполняется автоматически; для безлимитных — не отображается)
- Кнопки: Сохранить, Отмена

---

### 6.6 Расписание (`ScheduleView`)
**Представления:**
- Недельная сетка (по часам и дням)
- Список занятий (фильтр по тренеру / типу занятия)

**Карточка занятия содержит:** Тип занятия, Тренер, Время, Свободных мест
**Двойной клик** открывает `ClassDialog`

---

### 6.7 Диалог занятия (`ClassDialog`)
**Вкладки:**
- **Основное:** Тип занятия, Тренер, Дата, Начало, Конец, Макс. участников, Статус
- **Участники:** Список записанных клиентов, кнопки «Добавить клиента», «Отметить посещение»

---

### 6.8 Журнал посещений (`AttendanceView`)
**Фильтры:** По дате, клиенту, занятию, тренеру
**DataGrid:** Клиент, Занятие, Тренер, Дата/время, Статус
**Кнопки:** Отметить, Отменить отметку

---

### 6.9 Справочник типов абонементов (`MembershipTypesView`)
**DataGrid:** Название, Длительность (дни), Кол-во посещений, Безлимит, Цена
**Кнопки:** Добавить, Редактировать, Архивировать

---

## 7. Нефункциональные требования

- **ОС:** Windows 10/11 (x64)
- **Минимальный объём RAM:** 512 МБ
- **Разрешение экрана:** от 1280×720
- **База данных:** PostgreSQL, строка подключения хранится в `appsettings.json`
- **Локализация:** русский язык интерфейса
- **Валидация данных:** на уровне ViewModel и Entity Framework Fluent API
- **Логирование:** Serilog, вывод в файл `logs/`
