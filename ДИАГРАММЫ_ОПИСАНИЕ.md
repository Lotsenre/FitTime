# Описания диаграмм для курсовой работы FitTime

---

## 1. USE CASE ДИАГРАММА (Диаграмма вариантов использования)

### Акторы (3 фигурки-человечка, расположены слева и справа от овалов):

**Слева:**
- **Администратор** (человечек с подписью «Администратор»)
- **Менеджер** (человечек с подписью «Менеджер»)

**Справа:**
- **Тренер** (человечек с подписью «Тренер»)

### Варианты использования (овалы внутри прямоугольника «ИС FitTime»):

**Общие (связаны со всеми тремя акторами):**
1. «Авторизация в системе»
2. «Просмотр панели управления (Dashboard)»

**Администратор (линии от человечка к овалам):**
3. «Управление пользователями» (создание, редактирование, блокировка)
4. «Сброс пароля пользователя»
5. «Управление тренерами» (добавление, редактирование)
6. «Управление типами абонементов» (создание, архивация)
7. «Просмотр всех отчётов»
8. «Экспорт отчётов в CSV»

**Менеджер (линии от человечка к овалам):**
9. «Управление клиентами» (добавление, редактирование, деактивация)
10. «Оформление абонемента клиенту»
11. «Управление расписанием» (создание/редактирование занятий)
12. «Отметка посещаемости»
13. «Просмотр отчётов» (посещаемость, продажи, истекающие)
14. «Экспорт отчётов в CSV»

**Тренер (линии от человечка к овалам):**
15. «Просмотр своего расписания»
16. «Просмотр списка участников занятия»

### Связи между вариантами использования:
- «Управление клиентами» --<<include>>--> «Оформление абонемента клиенту» (оформление абонемента подразумевает работу с клиентом)
- «Управление расписанием» --<<include>>--> «Просмотр списка участников занятия»
- «Просмотр отчётов» --<<extend>>--> «Экспорт отчётов в CSV» (экспорт — расширение отчётов)

### Визуальное расположение:
- Прямоугольник-рамка по центру с надписью сверху «ИС FitTime»
- Овалы расположены внутри рамки в 3 колонки по смыслу
- Акторы-человечки за пределами рамки
- Линии от акторов к овалам — сплошные

## 2. SEQUENCE DIAGRAM (Диаграмма последовательности — сценарий «Авторизация»)

### Участники (прямоугольники сверху, слева направо):

1. **:Пользователь** (актор-человечек)
2. **:LoginWindow** (окно авторизации)
3. **:LoginViewModel** (логика авторизации)
4. **:FitTimeDbContext** (контекст БД)
5. **:BCrypt** (библиотека хеширования)
6. **:ICurrentUserService** (сервис текущего пользователя)
7. **:MainWindow** (главное окно)

### Последовательность сообщений (стрелки сверху вниз):

1. **Пользователь → LoginWindow:** «Ввод логина и пароля»
2. **LoginWindow → LoginViewModel:** «LoginCommand(login, password)»
3. **LoginViewModel → LoginViewModel:** «Проверка: поля не пустые»

   **[alt] Поля пустые:**
   - **LoginViewModel → LoginWindow:** «Показать ошибку: "Заполните все поля"»
   - (конец альтернативы)

4. **LoginViewModel → FitTimeDbContext:** «Users.FirstOrDefault(u => u.Login == login)»
5. **FitTimeDbContext → LoginViewModel:** «return user (или null)»

   **[alt] Пользователь не найден:**
   - **LoginViewModel → LoginWindow:** «Показать ошибку: "Неверный логин или пароль"»
   - (конец альтернативы)

6. **LoginViewModel → LoginViewModel:** «Проверка: user.IsActive && user.FailedAttempts < 5»

   **[alt] Аккаунт заблокирован:**
   - **LoginViewModel → LoginWindow:** «Показать ошибку: "Аккаунт заблокирован"»
   - (конец альтернативы)

7. **LoginViewModel → BCrypt:** «BCrypt.Verify(password, user.PasswordHash)»
8. **BCrypt → LoginViewModel:** «return true/false»

   **[alt] Пароль неверный:**
   - **LoginViewModel → FitTimeDbContext:** «user.FailedAttempts++; SaveChanges()»
   - **LoginViewModel → LoginWindow:** «Показать ошибку: "Неверный логин или пароль" + счётчик попыток»
   - (конец альтернативы)

9. **LoginViewModel → FitTimeDbContext:** «user.FailedAttempts = 0; user.LastLoginAt = DateTime.Now; SaveChanges()»
10. **LoginViewModel → ICurrentUserService:** «SetCurrentUser(user)»
11. **LoginViewModel → MainWindow:** «new MainWindow(); Show()»
12. **LoginViewModel → LoginWindow:** «Close()»

### Визуальные элементы:
- Прямоугольники участников сверху с пунктирными линиями жизни вниз
- Сплошные стрелки → для вызовов
- Пунктирные стрелки ← для возвратов
- Блоки [alt] — прямоугольники с пометкой «alt» в верхнем левом углу, разделённые пунктирной горизонтальной линией

## 3. ACTIVITY DIAGRAM (Диаграмма деятельности — процесс «Оформление абонемента»)

### Элементы (сверху вниз):

1. **● (чёрный круг)** — Начало

2. **▭ «Менеджер открывает раздел "Клиенты"»** — действие (прямоугольник со скруглёнными углами)

3. **▭ «Выбор клиента из списка»**

4. **▭ «Нажатие кнопки "Оформить абонемент"»**

5. **▭ «Открытие диалога оформления абонемента»**

6. **▭ «Выбор типа абонемента из выпадающего списка»**

7. **▭ «Система автоматически заполняет: цена, длительность (дни), кол-во посещений, дата окончания»**

8. **◇ Ромб принятия решения:** «Тип абонемента — Безлимит?»
   - **[Да]** → «Поле "кол-во посещений" скрыто, IsUnlimited = true»
   - **[Нет]** → «Отображается поле "кол-во посещений"»
   - Оба пути сходятся ↓

9. **▭ «Менеджер указывает дату начала»**

10. **▭ «Система рассчитывает дату окончания = дата начала + duration_days»**

11. **▭ «Менеджер нажимает "Сохранить"»**

12. **◇ Ромб принятия решения:** «Все поля заполнены корректно?»
    - **[Нет]** → «Показать ошибку валидации» → возврат к п.9
    - **[Да]** ↓

13. **▭ «Создание записи Membership в БД (client_id, membership_type_id, start_date, end_date, price, sold_by_user_id)»**

14. **▭ «Обновление списка абонементов клиента»**

15. **▭ «Закрытие диалогового окна»**

16. **◉ (круг с точкой внутри)** — Конец

### Визуальное расположение:
- Вертикальная ориентация, сверху вниз
- Скруглённые прямоугольники для действий
- Ромбы для условий с подписями [Да]/[Нет] на выходящих стрелках
- Стрелки между элементами

## 5. ДИАГРАММА КЛАССОВ (Class Diagram)

### Слои приложения (разделить визуально по блокам/пакетам):

---

### ПАКЕТ «Models» (сущности БД):

**Класс Role**
- Свойства: Id: int, Name: string, Description: string?
- Коллекции: Users: ICollection<User>

**Класс User**
- Свойства: Id: int, Login: string, PasswordHash: string, RoleId: int, FirstName: string, LastName: string, Patronymic: string?, Phone: string?, Email: string?, Specialization: string?, IsActive: bool, FailedAttempts: short, LastLoginAt: DateTime?, CreatedAt: DateTime
- Навигация: Role: Role
- Вычисляемые: FullName: string «get», ShortName: string «get»

**Класс Client**
- Свойства: Id: int, FirstName: string, LastName: string, Patronymic: string?, BirthDate: DateOnly?, Phone: string?, Email: string?, Notes: string?, IsActive: bool, CreatedAt: DateTime, UpdatedAt: DateTime
- Коллекции: Memberships: ICollection<Membership>, Enrollments: ICollection<ClassEnrollment>, Attendances: ICollection<Attendance>
- Вычисляемые: FullName: string «get», ShortName: string «get», Initials: string «get»

**Класс MembershipType**
- Свойства: Id: int, Name: string, DurationDays: int, IsUnlimited: bool, VisitCount: int, Price: decimal, Description: string?, IsArchived: bool, CreatedAt: DateTime
- Коллекции: Memberships: ICollection<Membership>

**Класс Membership**
- Свойства: Id: int, ClientId: int, MembershipTypeId: int, StartDate: DateOnly, EndDate: DateOnly, IsUnlimited: bool, VisitsRemaining: int, IsActive: bool, SoldByUserId: int?, Price: decimal, Notes: string?, CreatedAt: DateTime
- Навигация: Client: Client, MembershipType: MembershipType, SoldByUser: User?

**Класс ClassType**
- Свойства: Id: int, Name: string, Description: string?, Color: string, IsActive: bool
- Коллекции: Classes: ICollection<Class>

**Класс Class**
- Свойства: Id: int, ClassTypeId: int, TrainerId: int, Room: string, StartTime: DateTime, EndTime: DateTime, MaxParticipants: int, Status: string, Notes: string?, CreatedByUserId: int?, CreatedAt: DateTime
- Навигация: ClassType: ClassType, Trainer: User, CreatedByUser: User?
- Коллекции: Enrollments: ICollection<ClassEnrollment>, Attendances: ICollection<Attendance>

**Класс ClassEnrollment**
- Свойства: Id: int, ClassId: int, ClientId: int, MembershipId: int?, EnrolledAt: DateTime, Status: string
- Навигация: Class: Class, Client: Client, Membership: Membership?

**Класс Attendance**
- Свойства: Id: int, ClassId: int, ClientId: int, MembershipId: int?, CheckedInAt: DateTime, CheckedInByUserId: int?, Status: string, Notes: string?
- Навигация: Class: Class, Client: Client, Membership: Membership?, CheckedInByUser: User?

### Связи между моделями:
- Role ──1:*──► User (Role.Users)
- User ──1:*──► Class (как Trainer)
- Client ──1:*──► Membership
- Client ──1:*──► ClassEnrollment
- Client ──1:*──► Attendance
- MembershipType ──1:*──► Membership
- Membership ──1:*──► ClassEnrollment
- Membership ──1:*──► Attendance
- ClassType ──1:*──► Class
- Class ──1:*──► ClassEnrollment
- Class ──1:*──► Attendance

### ПАКЕТ «ViewModels» (ключевые):

**Абстрактный класс BaseViewModel** (наследует ObservableObject из CommunityToolkit.Mvvm)
- Свойства: нет собственных
- Назначение: базовый класс для всех VM

**Класс LoginViewModel** (наследует BaseViewModel)
- Свойства: Login: string, Password: string, ErrorMessage: string, LoginAttempts: string
- Команды: LoginCommand: IRelayCommand

**Класс MainViewModel** (наследует BaseViewModel)
- Свойства: CurrentView: object, PageTitle: string, CurrentUser: User, UserInitials: string, UserRole: string, IsManagerOrAdmin: bool, IsAdmin: bool
- Команды: NavigateCommand: IRelayCommand<string>, LogoutCommand: IRelayCommand

**Класс DashboardViewModel** (наследует BaseViewModel)
- Свойства: ActiveClientsCount: int, TodayClassesCount: int, MonthSalesCount: int, ExpiringCount: int, WeeklyAttendance: список, TodayClasses: список, RecentSales: список, ExpiringMemberships: список

**Класс ClientsViewModel** (наследует BaseViewModel)
- Свойства: Clients: ObservableCollection<ClientDisplay>, SearchText: string, SelectedFilter: string, SelectedClient: ClientDisplay, TotalCount: int
- Команды: AddCommand, EditCommand, DeactivateCommand, OpenProfileCommand

**Класс ScheduleViewModel** (наследует BaseViewModel)
- Свойства: Classes: список, WeekStart: DateTime, WeekEnd: DateTime, IsWeekView: bool, SelectedClass: ScheduleClassDisplay
- Команды: PrevWeekCommand, NextWeekCommand, AddClassCommand, EditClassCommand

**Класс ReportsViewModel** (наследует BaseViewModel)
- Свойства: SelectedTab: int, StartDate: DateTime, EndDate: DateTime, WeeklyAttendance: список, TopClasses: список, SalesData: список
- Команды: ApplyFilterCommand, ExportCsvCommand

### Связи ViewModels:
- Все VM ──наследование──► BaseViewModel
- MainViewModel ──композиция──► (все остальные VM через CurrentView)
- Все VM ──зависимость──► FitTimeDbContext
- Все VM ──зависимость──► ICurrentUserService
