# Project Instructions — FitTime

## 1. Самостоятельность и принятие решений
- Любые действия по созданию, изменению, удалению файлов (`.cs`, `.xaml`, `.csproj`, `.py`, `.sql` и др.) выполняются **без запроса подтверждения** у пользователя.
- Не тормозить разработку — просто делать.

## 2. Консольные команды
- Безопасные команды (`cd`, `dotnet run`, `dotnet clean`, `dotnet build`, `mkdir`, `ls` и т.п.) выполняются **без запроса разрешения**.
- Быть внимательным: деструктивные команды (удаление директорий, force-операции) всё же требуют осторожности.

## 3. Работа с БД (PostgreSQL)
- СУБД: **PostgreSQL 18**
- Пароль: `root`
- Пользователь: `postgres` (по умолчанию)
- Полный доступ: создание, изменение, удаление таблиц и данных — **без запроса подтверждения**.

## 4. Планирование сложных задач
- Любая сложная задача разбивается на подпункты.
- План записывается во внутреннюю память проекта (memory).
- После выполнения каждого крупного пункта — обновлять состояние плана в памяти.

## 5. Проверка после выполнения
- После завершения плана — **двукратная проверка** написанного кода:
  1. Ручная проверка логики и корректности.
  2. Code review через встроенный плагин (`/simplify` или code-reviewer agent).

## 6. Действия, НЕ требующие разрешения пользователя
Следующие операции выполняются **автоматически без подтверждения**:
- **Bash**: все безопасные команды — `ls`, `cd`, `mkdir`, `dotnet run`, `dotnet build`, `dotnet clean`, `psql` (любые SQL-запросы к PostgreSQL), `cat`, `echo`, `cp`, `mv`, `git status`, `git add`, `git diff`, `git log` и аналогичные
- **Write**: создание любых файлов проекта (`.cs`, `.xaml`, `.csproj`, `.sql`, `.py`, `.md`, `.json`, `.xml` и др.)
- **Edit**: редактирование любых файлов проекта
- **Read**: чтение любых файлов
- **Glob/Grep**: поиск файлов и содержимого
